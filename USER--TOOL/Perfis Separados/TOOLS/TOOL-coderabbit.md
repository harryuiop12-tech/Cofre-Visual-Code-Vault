# Ferramenta: CodeRabbit

Finalidade: revisar alterações de código com foco em bugs, regressões, segurança, qualidade e testes.

Quando acionar:
- Pedido de revisão de código.
- Feedback de pull request.
- Verificação de riscos antes de merge.
- Ciclo de correção após comentários técnicos.

Como agir:
- Priorizar achados por severidade.
- Referenciar arquivo e linha quando possível.
- Separar bug real, risco, lacuna de teste e preferência de estilo.
- Manter resumo curto depois dos achados.

Limite: não tratar revisão como elogio geral; a utilidade está em encontrar problemas acionáveis.
